Module Hierarchy

  • cocosCairo
    • cocosCairo.AbstractController: The controller in the model-view-controller design pattern.
    • cocosCairo.AbstractListener: Base class for Listeners to be used in conjunction with ListenedObjects.
    • cocosCairo.AbstractModel: The model in the model-view-controller design pattern.
    • cocosCairo.AbstractWindow
    • cocosCairo.Action: Basic Actions which can be performed on Nodes.
    • cocosCairo.ActionManager: Manages all Actions for the application.
    • cocosCairo.Color: Defines an RGBA color.
    • cocosCairo.ColorNode: A Node which fills its bounding box with a given Color.
    • cocosCairo.Director: The central part of the application.
    • cocosCairo.EaseAction: Actions which modifies the rate at which other Actions execute, usually by some geometric curve.
    • cocosCairo.GTKInterface: Provides an interface between cocosCairo and PyGTK.
    • cocosCairo.GTKWindow
    • cocosCairo.Geometry: Defines basic Geometry classes like Point, Rect, or Polygon as well as helper geometric functions.
    • cocosCairo.GestureDispatch: A ListenedObject that is responsible for sending user gestures to handlers.
    • cocosCairo.GestureEvent: An object that contains information about a detected gesture.
    • cocosCairo.GestureListener: A mixin that equips an object with handling Gesture events.
    • cocosCairo.InstantAction: Actions that execute once instantaneously.
    • cocosCairo.IntervalAction: Defines actions such as MoveTo or FadeOut which incrementally perform small changes in intervals over a period of time.
    • cocosCairo.Label: A Node which renders text to the screen.
    • cocosCairo.ListenedObject: A mixin which provides listenable methods to an object.
    • cocosCairo.Node: The base class for rendering items to the screen.
    • cocosCairo.PangoLabel: A Node which renders text to the screen using the Pango library.
    • cocosCairo.ParallaxNode: Creates a parallax effect.
    • cocosCairo.Primitive: Defines Nodes which display primitive geometric items, such as Polygons or Points, to the screen.
    • cocosCairo.SVGSprite: Renders SVG files to the screen.
    • cocosCairo.Scene: The main type of Node to which other Nodes are added.
    • cocosCairo.Scheduler: Manages all Timers for the application.
    • cocosCairo.Sprite: A Node that renders an image such as a JPEG to the screen.
    • cocosCairo.Timer: Calls a given callback every given seconds.
    • cocosCairo.Transition: A type of Scene that provides a transition from the currently-running Scene to a new Scene.
    • cocosCairo.cocosCairo: Convenience module which will import all cocosCairo modules into some other module (usually the main application).
